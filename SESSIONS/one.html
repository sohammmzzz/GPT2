Prerequisites

What is a Language Model?

A language model learns to predict the next token given a sequence of preceding tokens.

The solar panel converts → light (predicted)
  • Autoregressive — each prediction feeds back as input for the next
  • No supervision needed — the "label" is just the next word in the text
  • Emergent capability — predict well enough and the model learns grammar, facts, reasoning

Tokenization

Text → Tokens → IDs

Neural networks need numbers, not characters. We split text into sub-word pieces and map each to an integer.

"The solar panel converts light"
↓
The solar panel converts light
↓
464 38765 7115 26161 1657
GPT-2 vocabulary has 50,257 possible tokens — every integer is an index into this lookup table
BPE · Byte Pair Encoding

How tiktoken Splits Text

🔗 BPE Algorithm

  • Start with individual characters
  • Merge the most frequent adjacent pair
  • Repeat until vocab size is reached
  • Result: common words = 1 token; rare words = split

💡 Why sub-words?

"solar" → 1 token
"photovoltaic" → ["photo","volt","aic"]
"xkcd" → ["x","k","c","d"]

Handles any word with a fixed-size vocabulary

import tiktoken

tokenizer = tiktoken.get_encoding("gpt2")
# vocab size: 50,257
tokenizer.encode("solar")   # [38765]
tokenizer.encode("photovoltaic") # [5245,3020,3970]

Dataset Construction

The Sliding Window

We create training pairs by sliding a window across the token sequence. Input and target are offset by exactly 1 token.

■ input (max_length tokens) · ■ target (shifted +1)
Why offset by 1? — The model learns: "given tokens 0..N-1, what is token N?" Each window gives us one next-token prediction.
Code Walkthrough

GPTDatasetV1

class GPTDatasetV1(Dataset):
    def __init__(self, txt, tokenizer, max_length, stride):
        token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

        for i in range(0, len(token_ids) - max_length, stride):
            # input:  tokens[i .. i+max_length]
            self.input_ids.append(torch.tensor(token_ids[i : i+max_length]))
            # target: tokens[i+1 .. i+max_length+1]  ← shifted by 1
            self.target_ids.append(torch.tensor(token_ids[i+1 : i+max_length+1]))

def create_dataloader_v1(txt, batch_size=4, max_length=256, stride=128, ...):
    dataset   = GPTDatasetV1(txt, tokenizer, max_length, stride)
    dataloader = DataLoader(dataset, batch_size=batch_size, shuffle=shuffle)
    return dataloader
max_length

Context window — how many tokens the model sees at once

stride

Step size. stride < max_length creates overlapping windows → more training samples

DataLoader

Batches samples, shuffles, drops incomplete last batch

Token Embeddings

An integer index can't be passed to a neural network. We map each token ID to a trainable dense vector — a row lookup in a weight matrix.

token id

→
nn.Embedding(50257, 256)
256-dim vector
token_embedding_layer = torch.nn.Embedding(VOCAB_SIZE, EMBED_DIM)
# Shape: (50257, 256) — one row per token in the vocab

token_embeddings = token_embedding_layer(inputs)
# inputs shape:           (8, 4)        ← batch × seq
# token_embeddings shape: (8, 4, 256)   ← batch × seq × embed_dim
Embeddings

Positional Embeddings

Attention has no built-in sense of order. We inject position information by adding a learned position vector to each token.

Without position info

The model sees the same token embedding for "solar" whether it's at position 0 or position 100.

"solar" pos 0 = "solar" pos 5

With position info

pos 0 → unique vector p0
pos 1 → unique vector p1
pos 2 → unique vector p2
# Position embedding: one vector per slot
pos_emb_layer = nn.Embedding(CONTEXT_LENGTH, EMBED_DIM)

pos_embeddings = pos_emb_layer(
    torch.arange(CONTEXT_LENGTH)
)
# shape: (4, 256)  ← position × embed_dim

# Final input: token + position (same dim → add)
input_embeddings = token_embeddings + pos_embeddings
# shape: (8, 4, 256)
Both token & position embeddings have the same 256-dim shape — we simply add them element-wise

Self-Attention

Teaching tokens to look at each other and decide what information matters

📄 SELF_ATTENTION.py
Motivation

Why Does Context Matter?

She went to the river bank to fish

bank = riverbank

He deposited cash at the bank

bank = financial

The word "bank" has the same token ID in both sentences, but a different meaning. Self-attention lets each token look at all other tokens and update its representation based on context.

Self-attention computes a weighted blend of all token vectors — weights determined by how relevant each token is to the current one.

Self-Attention

Query · Key · Value

🔍
Query (Q)

"What am I looking for?"
Current token's question

🔑
Key (K)

"What do I represent?"
Each token's label

📦
Value (V)

"What do I carry?"
The actual information to share

📚 Dictionary analogy

Like a fuzzy dictionary lookup: your query is matched against all keys. The match score determines how much of each value you retrieve. More similar Q·K → more of that V flows through.

Q = x · W_q  ·  K = x · W_k  ·  V = x · W_v
Scaled Dot-Product

Computing Attention

x (B, T, d_in)
↓
W_q
Q = x·W_q
W_k
K = x·W_k
W_v
V = x·W_v
Q @ KT ÷ √d_head softmax × V
context = softmax(Q·KT / √d) · V
Why √d? — Dot products grow with dimension. Dividing by √d_head keeps the softmax from saturating to near-zero gradients.
Code Walkthrough

The SelfAttention Class

class SelfAttention(nn.Module):
    def __init__(self, d_in, d_out):
        super().__init__()
        self.W_query = nn.Linear(d_in, d_out, bias=False)  # learnable W_q
        self.W_key   = nn.Linear(d_in, d_out, bias=False)  # learnable W_k
        self.W_value = nn.Linear(d_in, d_out, bias=False)  # learnable W_v

    def forward(self, x):
        Q = self.W_query(x)                         # (B, T, d_out)
        K = self.W_key(x)                           # (B, T, d_out)
        V = self.W_value(x)                         # (B, T, d_out)

        scale        = K.shape[-1] ** 0.5          # √d_out
        attn_scores  = (Q @ K.transpose(-2, -1)) / scale  # (B,T,T)
        attn_weights = torch.softmax(attn_scores, dim=-1)   # rows sum to 1

        context_vecs = attn_weights @ V              # (B, T, d_out)
        return context_vecs, attn_weights
No bias=False

GPT convention — projection without bias, regularises the representation

(B,T,T) scores

Every token vs every token — T² comparisons per layer

Training

Training the QKV Weights

attn      = SelfAttention(D_IN, D_OUT)
optimizer = torch.optim.Adam(attn.parameters(), lr=1e-3)

for step in range(3):
    optimizer.zero_grad()
    context_vecs, _ = attn(x)      # forward pass
    loss = context_vecs.pow(2).mean()  # dummy loss
    loss.backward()               # gradients flow into W_q, W_k, W_v
    optimizer.step()              # update weights
  • .backward() traces the computation graph back through softmax, matmul, all three projections
  • W_q, W_k, W_v each get a gradient — the model learns what to query, what to key, and what to pass as value
  • In the full model, the loss is cross-entropy on next-token prediction — not a dummy loss
